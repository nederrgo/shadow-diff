# Monarch Repository Overview

This is a KubeBuilder-based Kubernetes Operator project for the `ShadowTest` CRD (`engine.shadow-diff.io/v1alpha1`).

It lives in the Shadow-Diff monorepo at `monarch/` (sibling to `beru/` and `igris/`). Run `make` from the repo root or from this directory.

## High-Level Layout

```text
<repo-root>/
  beru/                # Beru gRPC differ (separate Go module)
  igris/               # Igris HTTP multicaster (separate Go module, Phase 3a)
  monarch/             # This operator
    api/               # Kubernetes API (CRD Go types)
    cmd/               # Entrypoint (manager bootstrap)
    internal/controller/
    config/
    test/
    hack/
```

## Key Directories and Files

### `api/v1alpha1/`

- `shadowtest_types.go`
  - Defines `ShadowTestSpec` and `ShadowTestStatus`.
  - This is where CRD fields and validation markers live.
- `groupversion_info.go`
  - Registers Group/Version and scheme builder.
- `zz_generated.deepcopy.go`
  - Auto-generated deepcopy methods (do not edit manually).

### `internal/controller/`

- `shadowtest_controller.go`
  - Main reconcile entrypoint (`Reconcile`) + controller registration (`SetupWithManager`).
- `shadowtest_resources.go`
  - Resource reconciliation helpers:
  - namespace creation/cleanup, deployment create-or-patch, status patching.
- `shadowtest_helpers.go`
  - Pure helper functions (name sanitization, labels, env extraction).
- `shadowtest_constants.go`
  - Shared constants (labels, roles, finalizer).
- `shadowtest_controller_test.go`
  - Reconciler behavior tests.
- `suite_test.go`
  - envtest bootstrapping for controller tests.

### `cmd/`

- `main.go`
  - Starts controller manager.
  - Registers schemes and wires `ShadowTestReconciler`.
  - Configures health checks, metrics, leader election flags.

### `config/`

- `crd/`
  - CRD manifests and kustomization glue.
  - `config/crd/bases/engine.shadow-diff.io_shadowtests.yaml` is generated from API markers.
- `rbac/`
  - ServiceAccount, ClusterRole, RoleBinding, leader-election and metrics RBAC.
  - `role.yaml` is generated from `+kubebuilder:rbac` markers.
- `manager/`
  - Manager Deployment manifest.
- `default/`
  - Top-level install kustomization that ties CRD + RBAC + manager together.
- `samples/`
  - Example `ShadowTest` custom resource to apply and test quickly.
- `prometheus/`, `network-policy/`
  - Optional monitoring and network policy manifests.

### Root Build/Project Files

- `PROJECT`
  - KubeBuilder project metadata (plugins, layout, resources).
- `Makefile`
  - Standard tasks (`manifests`, `generate`, `test`, `run`, `deploy`, etc.).
- `go.mod` / `go.sum`
  - Go module dependencies.
- `Dockerfile`
  - Container image build for manager.
- `README.md`
  - User-facing docs (currently scaffolded, can be expanded).

## What Is Generated vs Hand-Written

### Mostly generated by KubeBuilder/controller-gen (avoid manual edits)

- `api/v1alpha1/zz_generated.deepcopy.go`
- `config/crd/bases/*`
- portions of `config/rbac/*` (from RBAC markers)

### Primary hand-written code

- `api/v1alpha1/shadowtest_types.go` (CRD fields/markers)
- `internal/controller/*.go` (business logic)
- `config/samples/*.yaml` (example CRs)

## Typical Development Flow

1. Update CRD types in `api/v1alpha1/shadowtest_types.go`.
2. Regenerate artifacts:
   - `make manifests`
   - `make generate`
3. Implement/adjust reconcile logic in `internal/controller/`.
4. Run tests:
   - `make test`
5. Run locally:
   - `make run`
6. Deploy to cluster:
   - `make deploy IMG=<your-image>`

## Monarch-Specific Behavior (Current)

- Watches `ShadowTest` resources.
- Resolves a target Deployment (`spec.targetDeployment` in `spec.targetNamespace`).
- Creates a dedicated shadow namespace.
- Creates/updates 3 Deployments:
  - `<name>-control-a`
  - `<name>-control-b`
  - `<name>-candidate`
- Copies **literal env vars only** from target primary container (MVP behavior).
- Uses finalizer cleanup for shadow namespace lifecycle.

## Sibling services (not in this directory)

| Service | Path | Role |
|---------|------|------|
| **Beru** | `beru/` | gRPC ingest + diff-of-diffs; Envoy sidecars report via `ext_proc` |
| **Igris** | `igris/` | Modular traffic engine (core + HTTP add-on) — Monarch deploys it per ShadowTest |

Monarch provisions shadow Deployments, Envoy config, shadow Services, and Igris (`listeners.json` from `spec.inputs`). See [ARCHITECTURE.md](../ARCHITECTURE.md).
